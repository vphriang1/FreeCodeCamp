Ninjutsu

One way to perform a substitution jutsu on a data is by storing the data in a temporary variable. For instance if we start off with var usedSubstitution = false; var ninja = 'naruto'; var temp = naruto; usedSubstitution = true; and an enemyAttacks function comes along ninja = enemyAttacks(ninja); function enemyAttacks(victim) { victim = null; return victim;}; the string naruto seems almost lost forever. But wait, we saved 'naruto' in a temp variable before the enemy function was able to destroy our 'naruto' string data forever. So actually the string 'naruto' still exists in our code. He's stored in the variable temp. And he still has a chance to be re-instantiated into any string type variable. Since var ninja was stored before he was destroyed, he has a chance to be used in his scope of the code. Thus if(usedSubstitution){ninja=temp}; is the next line of code our 'naruto' string still exists. Data should be stored if they should be reused again.

We also have shadow clone jutsu. If we have an array called var ninja = [10]; we could create many 'naruto' in our program by using the .fill() method. For instance, if we do ninja.fill('naruto', this built in javascript function will populate the the whole array with the same string: 'naruto'.
